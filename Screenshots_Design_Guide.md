## Advanced: Screenshots with Captions/Marketing Text

### Option A: Clean Screenshots (Recommended for Beginners)
Just capture the app exactly as it appears. Let the UI speak for itself.

**Pros:**
- Authentic representation
- Quick to create
- No design skills needed
- Apple prefers authentic screenshots

**Cons:**
- Less "pop" in crowded App Store
- Harder to communicate features quickly

### Option B: Screenshots with Text Overlays (More Work)
Add marketing text and graphics on top of screenshots.

**Pros:**
- Eye-catching
- Communicates value proposition clearly
- Can highlight specific features
- Professional marketing appearance

**Cons:**
- Requires design tools (Photoshop, Figma, Canva)
- More time-consuming
- Must comply with Apple guidelines (no misleading content)

**Tools for Text Overlays:**
- **Figma** (free, web-based)
- **Canva** (free tier available)
- **Adobe Photoshop** (professional)
- **Sketch** (macOS only)
- **Screenshot Creator** apps (various App Store tools)

**Text Overlay Guidelines:**
- Keep text large and readable
- Use high contrast (white text on dark overlay, or vice versa)
- Maximum 1-2 short phrases per screenshot
- Don't obscure important UI elements
- Maintain consistent style across all screenshots

## Status Bar Best Practices

### The "Apple Standard" Status Bar
Apple recommends a clean status bar:
- **Time:** 9:41 AM (Apple's signature time)
- **Cellular:** Full signal (5 bars or full dots)
- **WiFi:** Connected (full bars)
- **Battery:** Full or charging (100%, no red)
- **Bluetooth:** Hidden (unless relevant to app)
- **Location:** Hidden (unless relevant to app)

### How to Get Clean Status Bar in Simulator

#### Xcode Built-In (Easiest):
Xcode Simulator automatically shows clean status bar by default.

## Screenshot Checklist

### Before Taking Screenshots
- [ ] App builds and runs without crashes
- [ ] UI looks polished (no placeholder text)
- [ ] No debug text or developer tools visible
- [ ] Game is in interesting state (not just Week 1, Turn 1)
- [ ] Funds are not $0 (looks better with activity)
- [ ] Map has variety (not all one color)
- [ ] Status bar is clean (or will be edited)

### Taking Screenshots
- [ ] Correct device selected (iPhone 15 Pro Max, iPad Pro 12.9")
- [ ] Correct orientation (portrait recommended)
- [ ] All 5-6 screenshots captured
- [ ] Screenshots show variety (different screens/features)
- [ ] Screenshots tell a story (setup ‚Üí gameplay ‚Üí results)
- [ ] No duplicate or very similar screenshots

### After Taking Screenshots
- [ ] Files saved in accessible location
- [ ] File names descriptive (e.g., "01_Hero.png", "02_Map.png")
- [ ] Dimensions verified (exact pixels required)
- [ ] Format verified (PNG or JPEG accepted, PNG recommended)
- [ ] File size reasonable (<10MB each)
- [ ] Visual quality checked (no pixelation, blur)
- [ ] Text is readable at full size
- [ ] Colors accurate to app

### For iPad Screenshots
- [ ] UI adapts properly to iPad layout
- [ ] Not just scaled-up iPhone version
- [ ] Takes advantage of larger screen
- [ ] All elements visible and properly sized

### Before Uploading to App Store Connect
- [ ] Have both iPhone and iPad sets
- [ ] 5-6 screenshots for each device size
- [ ] Screenshots ordered intentionally (best first)
- [ ] No personal information visible
- [ ] No offensive or inappropriate content
- [ ] Screenshots match app's actual appearance
- [ ] Tested viewing on different displays
